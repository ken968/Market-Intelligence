1. DAILY INFERENCE EXECUTION PIPELINE
The daily stochastic inference loop is orchestrated by `utils/predictor_engine.py` and `utils/layers/manager_anchor.py`. The execution path follows strict state-machine logic:
- State 1 (Ingestion): Fetch $t=0$ closing prices and calculate deterministic technical indicators.
- State 2 (Macro Sync): Pull current state from DuckDB OLAP tables, calculating synthetic divergence metrics (e.g., Smart Money Gap) on the full historical dataset to guarantee valid `.rolling()` window evaluations before extracting the $t=0$ row.
- State 3 (Base Inference): Execute LSTM forward pass on the scaled 90-day feature window; execute XGBoost forward pass on the single macro vector.
- State 4 (Meta Inference): Feed Base Learner outputs + Real-time Context (VIX, DXY, GK Volatility) into the Dual-Head Stacker.

2. TRAJECTORY GENERATION: POWER-LAW MOMENTUM DECAY
The system must generate a 90-day forward price curve from a single 7-Day Anchor target. Extrapolating a 7-day return linearly to 90 days would violate the Efficient Market Hypothesis (EMH) and mean-reverting properties of speculative assets.
Instead, the system utilizes a Power-Law Momentum Decay equation:
$$ R_i = R_{\text{anchor}} \times \left( \frac{i}{7.0} \right)^\alpha $$
Where:
- $R_i$ = Projected cumulative percentage return at day $i$.
- $R_{\text{anchor}}$ = The 7-day expected return from the Dual-Head Stacker.
- $\alpha = 0.65$ = The decay exponent.
Because $\alpha < 1$, the trajectory exhibits momentum clustering in the short term (rapid initial movement) but experiences asymptotic decay in the long term, simulating the exhaustion of directional buying/selling pressure. This ensures mathematical coherence between the tabular data projection and the graphical UI representation.

3. RETRAINING CYCLE & CONCEPT DRIFT MITIGATION
To mitigate Concept Drift (the statistical degradation of a model's predictive power as macroeconomic regimes shift), the system mandates a periodic retraining protocol. Given that the dominant Institutional Smart Money variable (CFTC COT) operates on a weekly cadence (Friday release), daily retraining introduces unnecessary computational overhead and risks stochastic overfitting. The retraining cycle is strictly scheduled post-COT ingestion.

4. PHASE 7 ROADMAP: EXECUTION & RISK LAYER INTEGRATION
The ultimate maturation of the Market Intelligence Terminal lies in its transition to a probabilistic risk manager.
- Dynamic Kelly Sizing: The `direction_prob` output from the Directional Head directly feeds the probability variable ($P$) in the Kelly Criterion formula: $f^* = P - \frac{1-P}{B}$. This transitions the system from providing "Buy/Sell" signals to providing mathematically optimal portfolio exposure sizing.
- Volatility-Adjusted Confidence Bands: Integrating `GK_Vol_21d` to dynamically widen or tighten projection confidence intervals, allowing users to set dynamic Stop-Loss thresholds that breathe with current market liquidity rather than static percentage assumptions.
